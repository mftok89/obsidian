## 0. たどり着いた経緯（前提）

- ブロックチェーンの安全性には**2つの層**がある。
  - **根幹（暗号の数学そのもの）** … ハッシュ関数・電子署名など。計算量の壁に守られている。
  - **応用（スマートコントラクト・運用）** … 実際のハッキング被害の大半はこちら。一般のIT・金融セキュリティと地続き。
- **AIは「根幹の暗号」を直接破る存在ではない**。AIが効くのは主に応用層（脆弱性発見・監査・詐欺など、攻防両面）。
- 根幹の暗号を脅かす**本命は量子コンピューター**。← だからここを深掘りする。

> [!note] ポイント
> 「根幹は当面安泰／脅かされるのは実装と運用の層」。ただしブロックチェーンは**取引を取り消せない**ぶん、同じハッキングでも被害が重くなりやすい。

## 1. 普通のコンピューターとの違い：ビットと量子ビット

| | 普通のコンピューター | 量子コンピューター |
|---|---|---|
| 最小単位 | ビット | 量子ビット（キュービット） |
| 状態 | 0 か 1 の**どちらか一方** | 0 と 1 が**混ざった状態**も取れる |
| たとえ | テーブルに置いたコイン（表か裏で確定） | **指で弾いて回転中のコイン** |

---

## 2. 重ね合わせ｜「回転するコイン」

- **重ね合わせ（superposition）** = 0でも1でもない「両方が混ざった状態」。
- たとえ：**回転中のコイン**。クルクル回っている間は表とも裏とも言いきれない。
- **測定（観測）した瞬間に、0か1のどちらかにパタンと確定する**。回転中の混ざった状態そのものは直接覗けない。

> [!warning] よくある誤解
> 「0と1を同時に持つ＝答えが全部同時にわかる」は**半分ミスリード**。最後に測定したら結局1つの答えしか出てこない。「混ざった状態をうまく使う」ことと「答えを全部取り出せる」ことは別物。

---

## 3. 干渉｜間違いを消し、正解を強める仕掛け

- 重ね合わせには「0寄り／1寄り」の**重み**がついている。量子計算とは、突き詰めるとこの重みを操作する作業。
- ただしこの重み（**確率振幅**）は、ただの確率と違って**プラスにもマイナスにもなれる**。ここが決定的。

### なぜマイナスが嬉しいのか → 波のたとえ

- 山と山が重なる → **もっと大きな山**（強め合う）
- 山と谷が重なる → **打ち消し合って平ら**（弱め合う）
- これが **干渉（interference）**。

> [!tip] 量子コンピューターの核心の仕掛け
> たくさんの可能性を重ね合わせておき、
> **「間違った答え」の重みどうしを打ち消し合って消し、「正しい答え」の重みだけを強め合わせる。**
> → 最後に測定すると、正しい答えが高い確率で出てくる。

- だから「ただ全部試す」だけでは無意味（測定でランダムに1つ出るだけ）。**いらない答えを干渉で消す設計**ができて初めて、普通のコンピューターに勝てる。

---

## 4. デコヒーレンス｜最大の敵

- 干渉を成立させるには、**全部の量子ビットが「回転中の混ざった状態」を計算終了まで保ち続ける**必要がある。
- 途中で1個でも止まる（確定する）と、波の性質を失い、計算が崩れる。

> [!danger] デコヒーレンス（decoherence）
> 「混ざった状態がうっかり壊れてしまう現象」。
> 厄介なのは、人間が覗かなくても**外界と少しでも関わるだけで実質“測定”になり、勝手に壊れる**こと。
> 原因例：わずかな**熱**・**振動**・**電磁波**・周囲の原子の衝突 など。

- たとえ：**超繊細なコマ**。完璧な静寂でしか美しく回れず、回り続けられるのは一瞬（マイクロ秒〜ミリ秒）。
- だから実機はあの**シャンデリアみたいな金色の装置**になる。大部分は隔離装置。
  - **絶対零度近く**（-273℃近く、宇宙より冷たい）まで冷却 → 熱のゆらぎを消す
  - 振動・電磁波を**徹底遮断**
- 「速い計算」以前に「**繊細な状態を壊さず保つ**」だけで巨大装置と莫大コスト。← 実用化が遅い最大の理由。

### 核心の3点セット

1. **重ね合わせ** … 0と1が混ざった「回転するコイン」
2. **干渉** … プラス・マイナスの重みで、間違いを消し正解を強める
3. **デコヒーレンス** … その繊細な状態がうっかり壊れる、最大の敵

## 5. 今どこまで実現できているか（2026年6月時点）

> [!summary] 結論
> **すごく進んでいるが、暗号を破れる地点にはまだ全然届いていない。**

### 「数」は増えている

- 2026年3月、理研が144量子ビット「叡II」のクラウド提供を開始。
- 富士通・理研は2026年度内に**1,000量子ビット機**を目指す。2030年には1万量子ビット超が目標。

### でも本質は「数」より「質」へ

- デコヒーレンス＝**そのまま計算の間違い（エラー）**。量子ビットは放っておくとすぐ間違える。
- 2026年は「数を並べる量的競争」から「**エラーを防ぐ質的競争**」へ転換。
- カギは **誤り訂正（エラー訂正）**：
  - **予備の量子ビットを大量に束ねて、信頼できる1個分**を作る発想。
  - 倒れやすいコマ1個のために、周りに何十個も見張り役を置いて支え合わせるイメージ。
  - Googleの「Willow」（2024/12）が「量子ビットを増やすほどエラー率が下がる」閾値を実証 → 大きな一歩。
- この「束ねて作った信頼できる1個」= **論理量子ビット**。
  - IBMは**200論理ビット**級の誤り訂正マシンを**2028年まで**に開発する計画。
  - → 「ちゃんと信頼できる量子ビット」はまだ**数百個レベルにこれから到達するか**という段階。

### ロードマップ（用語整理つき）

- **量子優位性** … ある特殊な問題でスパコンより速い、を示す段階。IBMは**2026年末**目標。← 今まさに狙うところ。
- **誤り耐性（フォールトトレラント）量子コンピューター（FTQC）** … エラーを抑えて実用計算を安定して回せる本命。IBMは**2029年**目標。← まだ数年先。

> [!quote] 現在地のイメージ
> 「ものすごく繊細なコマを、1個まともに回し続ける技術がようやく形になってきた」段階。
> **ライト兄弟がやっと数百メートル飛べた**くらい。ジャンボジェット（暗号を破れる規模）には機数も安定性も全然足りない。

## 6. なぜ暗号への脅威なのか／なぜ「今」対策するのか

### どれくらいの規模が必要か

- 今広く使われるRSA暗号を破るには、ざっくり**数百万個の物理量子ビット**が必要と見積もられる。
- 今の最先端は千個レベル、信頼できる論理量子ビットは数百個がこれから。
- → **必要なのは数百万個、今あるのは千個レベル。差は1000倍以上。**
- しかも「束ねて1個」を考えると、実際の差はさらに開く。
- たとえ：ライト兄弟の直後に「月まではいつ?」と聞くようなもの。← だから「現時点では暗号は安泰」。

### なら、なぜ今から大騒ぎして対策するのか？

> [!important] 理由1：「今盗んで、後で解読する」攻撃
> **Harvest Now, Decrypt Later（今収穫し、後で復号する）**
> 攻撃者は暗号化データを**今盗んで保存**しておき、量子が完成した**10年後にまとめて解読**できる。
> → 「完成してから対策」では手遅れ。今流れている秘密（国家機密・医療・金融など、将来も秘密であってほしいもの）は**もう盗まれて寝かされているかも**。

- **理由2：暗号の引っ越しには膨大な時間がかかる**
  - 銀行・Web・スマホ・IoT…あらゆる場所に今の暗号が埋め込まれている。
  - 新暗号への総入れ替えは何年〜十年がかりの大事業 → 完成を待たず今から動く必要。

### 対策はすでに存在する → 耐量子暗号（PQC）

> [!success] 耐量子暗号（Post-Quantum Cryptography, PQC）
> 守る側は量子技術を使わなくていい。**普通のコンピューターで動く、「量子でも解くのが難しい新しい数学」ベースの暗号**を使えばよい。
> すでに標準仕様が定められ、実装・移行が始まっている。

## 7. 全体まとめ

最初のブロックチェーンの話から、こうつながる：

1. ブロックチェーンの**土台の暗号**を脅かす本命は、AIではなく**量子コンピューター**。
2. でもその量子は、暗号を破るにはまだ**1000倍以上スケールが足りず、当面は安泰**。
3. ただし **「今盗んで後で解読」** があるので、**脅威は未来でも対策は“今”**。
4. 幸い **耐量子暗号（PQC）** という解がすでにあり、世界は移行を始めている。

## 用語集

| 用語 | ざっくり意味 |
|---|---|
| 量子ビット（キュービット） | 0と1が混ざった状態も取れる量子版のビット。たとえは「回転中のコイン」 |
| 重ね合わせ（superposition） | 0でも1でもない、混ざった状態 |
| 確率振幅 | 量子の「重み」。プラスにもマイナスにもなれるのがミソ |
| 干渉（interference） | プラス・マイナスの重みが打ち消し／強め合う現象。間違いを消し正解を強める |
| デコヒーレンス（decoherence） | 繊細な混ざった状態が外界との接触でうっかり壊れる現象。最大の敵 |
| 論理量子ビット | 多数の物理量子ビットを束ねて作る「信頼できる1個」 |
| 誤り訂正 | 予備の量子ビットで間違いを検出・補正する技術 |
| 量子優位性 | 特殊な問題でスパコンを上回ることを示す段階 |
| FTQC（誤り耐性量子コンピューター） | エラーを抑え実用計算を安定して回せる本命。数年先 |
| Harvest Now, Decrypt Later | 今暗号データを盗んで保存し、将来量子で解読する攻撃 |
| 耐量子暗号（PQC） | 量子でも破りにくい、普通のPCで動く新しい暗号 |
