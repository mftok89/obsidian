# Claude Cowork で AI 秘書を作るノウハウガイド

## 概要

Claude Cowork を使うと、Gmail・Google カレンダー・Notion・Slack などの外部ツールと連携しながら、**手元のフォルダーに情報を蓄積して育てていける AI 秘書**が作れる。

通常の ChatGPT や Claude チャットとの最大の違いは **ローカルフォルダーに直接アクセスしてファイルを作成・編集できる点**。

---

## 1. 前提：なぜ Claude Cowork が必要か

| ツール | 外部ツール連携 | ローカルファイル操作 | 情報の蓄積・育成 |
|--------|--------------|------------------|----------------|
| ChatGPT / Claude（Web版） | ○ | × | △（チャット履歴のみ） |
| **Claude Cowork** | ○ | **○** | **○（フォルダーに資産化）** |

ポイント：毎回チャットが独立している通常版と違い、Cowork はフォルダー内にどんどん情報が溜まっていくため、**使えば使うほど賢くなる**。

---

## 2. フォルダー構成（PARA メソッド）

AI 秘書専用フォルダーを作り、以下の構成で管理する。

```
AI秘書/
├── CLAUDE.md              ← 最重要：AIへの指示書
├── Projects/              ← 進行中のプロジェクト
│   └── Project001/
│       ├── index.md       ← プロジェクトの進行状況メモ
│       └── outputs/       ← 成果物（PPTX・ドキュメント等）
├── Areas/                 ← 継続管理する領域（クライアント、会社情報、日々の業務）
├── Resources/             ← テンプレート・参考資料
├── Archive/               ← 使わなくなったものの保管
└── Contexts/              ← AI の記憶・学習情報
    ├── memory/            ← 学習事項・用語・設定・失敗パターン
    ├── people/            ← 関係者のプロフィール（例：田中.md）
    └── profile.md         ← 自分のプロフィール・返信スタイル等
```

**PARA メソッドとは：** Projects / Areas / Resources / Archive の4カテゴリで情報を整理する情報管理手法。

---

## 3. CLAUDE.md（最重要ファイル）

### 役割

Claude Cowork（およびClaude Code）が**起動時に最初に読み込む指示書**。AI 秘書の設計図。

- AI 秘書としての役割・人格の定義
- 出力ルール（言語・トーン等）
- フォルダー構成と各フォルダーの用途
- 運用ルール（例：新規依頼はかならずプロジェクトを立ち上げてからアウトプット作成）
- コンテキスト保存ルール（どの情報をどこに記録するか）

### 作成のコツ

- **200行以内を目標に、コンパクトに書く**（長すぎると AI が読まなくなる）
- 枝葉の情報は別ファイルに切り出し、CLAUDE.md はメインの指示に絞る
- 試行錯誤しながら**継続的にチューニング**していくことが大事
- 書けば書くほど良いわけではない。情報が混乱すると AI の判断が崩れる

### CLAUDE.md の基本構成（例）

```markdown
# AI秘書 指示書

## 役割
あなたは私の個人秘書です。スピード重視で雑用・資料作成・情報整理をサポートしてください。

## 出力ルール
- 日本語・カジュアルなトーンでOK
- 確認不要な場合はそのまま実行

## フォルダー構成
- Projects/ : 案件ごとにフォルダーを作成し、成果物はoutputs/に保存
- Contexts/memory/ : 学習事項・用語・失敗パターンを記録（同じミスを繰り返さない）
- Contexts/people/ : 関係者情報を人物名.mdで記録

## 運用ルール
1. 新規依頼 → まずプロジェクトを立ち上げてからアウトプット作成
2. 既存案件の追加依頼 → 同じプロジェクトフォルダーに積み上げ
3. 関係者とやり取りがあったら Contexts/people/ に記録・更新

## 情報管理
PARAメソッドに基づいて情報を管理する。
参照：[PARAメソッドの説明]
```

## 4. コンテキスト（記憶）の仕組み

AI 秘書が「育つ」のは、Contexts フォルダーに情報が蓄積されるから。

| 保存場所 | 内容 |
|---------|------|
| `Contexts/memory/` | 用語、学習事項、失敗パターン、ツール設定 |
| `Contexts/people/` | 関係者の名前・メールアドレス・役割・過去のやり取り |
| `Contexts/profile.md` | 自分の情報、返信スタイル、好み |

**やり取りの後に「記録してください」と一言添えるだけで自動保存してくれる。**

## 5. コネクター（外部ツール連携）

Cowork の設定画面（カスタマイズ > コネクター）から連携する。

### 主な対応ツール

| ツール | できること |
|--------|-----------|
| **Gmail** | メール読み取り・下書き作成 |
| **Google カレンダー** | 予定の確認・登録・招待送付 |
| **Notion** | ページ読み取り・タスクDB追加 |
| **Slack** | メッセージ読み取り・送信 |
| Canva / Figma | デザイン作業 |
| HubSpot | CRM・営業管理 |
| Atlassian | Jira・Confluence 連携 |

### カスタムコネクター（MCP）

公式にないサービスは **MCP（Model Context Protocol）** を使って自作連携も可能。難易度は高め。まずは公式コネクターから始めるのがおすすめ。

MCP = AI 向けの USB 規格のようなもの。対応サービスか確認するには `サービス名 MCP` で検索。

## 6. プラグインとスキル

設定画面（カスタマイズ > プラグイン）から追加できる。

- **スキル**：特定の作業（PPTX作成・PDF操作・分析など）を AI に装備させる能力
- **プラグイン**：複数のスキルをまとめたパッケージ

### 主なプラグイン（Anthropic公式）

- **Finance** : 財務分析・レポート作成
- **Marketing** : マーケティング戦略・コンテンツ作成
- **Data** : データ分析・SQL・可視化
- **Operations / Productivity** : 業務効率化
- **Sales** : セールス支援

**→ 必要そうなプラグインを入れておくと、ゼロベースではなく専門知識を持った AI 秘書になる。**

---

## 7. スケジュールタスク（自動定期実行）

Cowork でスケジュール実行が可能。毎朝メールチェックや定期レポートを自動化できる。

### 設定手順

1. Cowork 画面の左サイドバー「予定済み」から新規タスクを追加
2. 実行フォルダー・モデル・頻度・時刻を設定
3. 指示文を入力（例：「届いているメールを確認して、返信が必要なものは下書きを作成。不要なものにはアーカイブラベルをつけてください」）
4. 保存

### 注意点

- **Claude Cowork のデスクトップアプリを起動したままにする必要がある**（スリープ状態ではタスクが動かない）
- 設定画面の「コンピューターをスリープさせない」をオンにしておく
- 時刻が若干ずれる場合があるが、バグの可能性あり（今後修正される見込み）

---

## 8. グローバル指示（任意）

設定 > グローバル指示 に自分の基本情報を書いておくと、どのフォルダーで Cowork を開いても共通の指示として機能する。

- 使い方：自分の役職・よく使うスタイル・基本情報など、すべてのプロジェクトに共通する内容を入れる
- ただし、AI 秘書専用フォルダーを固定して使う場合は CLAUDE.md で管理する方がシンプル

---

## 9. フォルダー選択の注意点

Cowork で開くフォルダーは **AI 秘書専用フォルダーだけ** にする。

デスクトップ全体や Downloads フォルダーを指定すると：
- 不要なファイルがコンテキストに入り込む
- AI の判断がぶれる
- 余計な情報を誤って操作するリスクがある

**→ AI 秘書フォルダーを作って、必要な情報だけをそこに集約していく習慣が大切。**

## 10. 他エージェントとの互換

| エージェント | 指示書ファイル名 |
|------------|----------------|
| Claude Cowork / Claude Code | `CLAUDE.md` |
| Cursor / Codex / その他 | `agent.md` |

書き方は同じ。Claude 系は `CLAUDE.md`、それ以外は `agent.md` が標準。

## 運用のコツ

1. **CLAUDE.md を育てる** - 「次からはこうして」という修正指示を CLAUDE.md に反映してもらう
2. **プロジェクト単位で管理** - 新しい依頼は必ずプロジェクトフォルダーを立ち上げてから
3. **コンテキストを溜める** - 関係者情報・用語・失敗パターンを都度記録
4. **コンパクトに保つ** - CLAUDE.md が肥大化したら定期的に整理・圧縮する
5. **外部ツールを連携させる** - Gmail+カレンダーだけでも、メール→下書き→カレンダー登録まで自動化できる
