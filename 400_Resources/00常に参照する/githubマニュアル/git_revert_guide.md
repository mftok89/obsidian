# 🕐 Gitで特定のコミットに戻る方法

> **対象読者:** Gitを使い始めたばかりの初心者の方  
> **所要時間:** 約10分で読めます

---

## そもそも「コミット」って何？

Gitでは、ファイルの変更を**セーブポイント**として保存できます。  
この「セーブポイント」のことを **コミット** と呼びます。

```
[最初の状態] → [コミット1] → [コミット2] → [コミット3（現在）]
                                ↑
                           ここに戻りたい！
```

コミットにはそれぞれ**コミットハッシュ**（識別番号）が割り当てられています。

---

## STEP 1: コミットの履歴を確認する

まず、どのコミットに戻りたいかを確認します。

```bash
git log --oneline
```

### 実行結果の例

```
a1b2c3d バグ修正：ログイン処理を修正
e4f5g6h 新機能：検索機能を追加     ← ここに戻りたい場合
i7j8k9l 初期コミット
```

- 左側の英数字（例: `e4f5g6h`）が **コミットハッシュ** です
- コミットは**新しい順**（上が最新）に表示されます

---

## 方法は2つ：状況に応じて選ぼう

| 状況 | 使うコマンド |
|------|------------|
| チーム開発・すでにGitHubへプッシュ済み | `git revert`（安全） |
| 自分のパソコンだけで作業中・まだプッシュしていない | `git reset`（強力） |

---

## 方法1: `git revert`（安全・推奨）

### どんな動作をするの？

「過去のコミットを打ち消す」新しいコミットを作ります。  
履歴は**消えず**に残るので、チーム開発でも安心して使えます。

```
[コミット1] → [コミット2] → [コミット3] → [コミット3を打ち消すコミット]
                                                      ↑
                                              ここが新しく追加される
```

### 使い方

```bash
git revert e4f5g6h
```

> `e4f5g6h` の部分は、戻したいコミットのハッシュに変更してください。

### 実行後の流れ

1. テキストエディタが開き、コミットメッセージの入力を求められます
2. そのまま保存（`:wq` または `Ctrl+S`）すれば完了です
3. `git log` で確認すると、新しいコミットが追加されています

### GitHubへ反映する場合

```bash
git push origin main
```

---

## 方法2: `git reset`（強力・慎重に使おう）

### どんな動作をするの？

指定したコミットまで**履歴ごと戻します**。  
過去の状態に完全に巻き戻すイメージです。

```
[コミット1] → [コミット2]    ※コミット3は消える！
                   ↑
              ここまで戻る
```

⚠️ **注意:** すでにGitHubにプッシュしている場合は、チームメンバーの作業を壊す可能性があります。

### 3つのオプション

#### `--soft`（変更をステージングに残す）

```bash
git reset --soft e4f5g6h
```

- ファイルの変更内容は**残ります**
- `git add` 済みの状態になります
- 「やっぱりコミットをやり直したい」ときに便利

#### `--mixed`（変更を作業ディレクトリに残す）※デフォルト

```bash
git reset --mixed e4f5g6h
# または単に
git reset e4f5g6h
```

- ファイルの変更内容は**残ります**
- `git add` する前の状態に戻ります
- 「変更は残しつつ、コミット履歴だけ消したい」ときに便利

#### `--hard`（変更を完全に破棄する）

```bash
git reset --hard e4f5g6h
```

- ファイルの変更内容も**すべて消えます**
- 元に戻せないので注意してください！
- 「完全にあの時点に戻したい」ときに使います

---

## オプション比較表

| オプション | ファイルの変更 | ステージング | コミット履歴 |
|-----------|------------|------------|-----------|
| `--soft`  | 残る        | 残る（add済み） | 削除 |
| `--mixed` | 残る        | 削除        | 削除 |
| `--hard`  | **削除**    | 削除        | 削除 |

## よくある質問

### Q. コミットハッシュは何文字入力すればいい？

最初の **7文字** で通常は十分です。

```bash
# フルのハッシュ（長い）
git revert e4f5g6h7i8j9k0l1m2n3o4p5q6r7s8

# 先頭7文字でもOK！
git revert e4f5g6h
```

### Q. 間違えて `--hard` してしまった！元に戻せる？

直後であれば `git reflog` で復元できる可能性があります。

```bash
git reflog
# 操作履歴が表示されるので、戻したい状態のハッシュを探す

git reset --hard <戻したいハッシュ>
```

### Q. `git revert` と `git reset` どちらを使えばいい？

迷ったら **`git revert`** を使いましょう！  
安全で、チーム開発でも問題なく使えます。

---

## まとめ

```
✅ チーム開発・プッシュ済み → git revert（安全）

🔧 ローカルのみ作業中
  ├─ 変更を残したい   → git reset --soft / --mixed
  └─ 完全に消したい   → git reset --hard（注意！）
```

**困ったら `git revert` を使っておけば安心です！**

---

> 📝 **参考:** コミットハッシュは `git log --oneline` で確認できます。
