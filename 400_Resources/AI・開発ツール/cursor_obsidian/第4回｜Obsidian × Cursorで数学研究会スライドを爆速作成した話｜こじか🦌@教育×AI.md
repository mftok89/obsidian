## 第4回｜Obsidian × Cursorで数学研究会スライドを爆速作成した話

[こじか🦌@教育×AI](https://note.com/n_t7627)

## はじめに

今年3月末、私は絶望していました。

**「3ヶ月後に数学研究会で登壇することになったけど、準備する時間なんてあるのか・・・」**

日中は学校で一番忙しい教務部の仕事。しかも、 **一年で最も忙しい年度初めという最悪のタイミング。** 高3担当の数学教師として、生徒の受験指導にも追われる。勤務時間内に発表準備をする時間なんて、1秒もありません

家に帰ったら帰ったで、また別の戦いが始まります。

定時ダッシュで学童へお迎え、帰宅後は食事の準備、年子の宿題チェック、家事、そして明日の授業の予習。すべてが片付く頃には寝る時間。

そんな日々を繰り返すうちに、あっという間に2ヶ月が経過。発表の日だけが、容赦なく近づいていました。

**しかし、あるツールとXポストとの出会いがすべてを変えました。**

Obsidian・Cursor・Marpを組み合わせることで、 **発表前日のたった1時間で、発表用スライドを完成させることができた** のです。

今回は、「Obsidian即メモ爆速アウトプット術」連載のきっかけとなったその驚きの体験と、実際のワークフローをお見せします。

---

## 📝 Day 1〜前日まで：発表に関する情報を蓄積

このワークフローで最も重要なのが、日々の情報蓄積です。

**最終的なアウトプットの質は、ここでほぼ決まります** 。どれだけ優れたツールを使っても、元となる情報が薄ければ、良いスライドは作れません。

私は1ヶ月間、自分のアイデアや気づき、実践記録、ブックマークなど、発表に関するあらゆる情報をObsidianに蓄積しました。

![画像](https://assets.st-note.com/img/1761366860-w3ktEDx1jdWK2eaT5iCvsBgM.png?width=1200)

日々のメモは一枚ノートで管理。目的別にタグも付けておくと、あとで再利用しやすい。

![画像](https://assets.st-note.com/img/1761369375-3qdzUH8FESAKCDNv92fTQy56.png?width=1200)

ブックマーク（Web Clipper使用）は目的別にフォルダ管理すると、あとで再利用しやすい。

**しかし、すぐに問題に直面しました。メモを取る作業そのものが、想像以上に面倒だったのです。**

この問題をどう解決したのか、詳しくはこちらのnoteで解説しています。

> Obsidian Web Clipperは有名なので、活用術noteは今のところ書いていません。使い方は調べればすぐに出てきます。

## ⚡️現在の運用：Obsidianノートをワンポチでスライド化

数学研究会での発表が終わった後、Obsidianプラグイン「Marp Slides」を発見。このプラグインを使えば、Obsidianノートをワンクリックでスライド化できます。

すべての作業がObsidian内で完結するため、さらなる時短に。

![画像](https://assets.st-note.com/img/1761388718-3m61fsxKZCrIMWEDgdavqujU.png?width=1200)

恩人・すぅさんが運営するPM研究所で、外部講師としてObsidian勉強会を行った際のスライド

手順は以下のとおりです。

### STEP1：プラグイン「Marp Slides」をインストール

![画像](https://assets.st-note.com/img/1761390220-diESo0TGKVD4eQ9Izaygwt8p.png?width=1200)

プラグインのインストール方法はこちらのnoteで解説しています。

### STEP2：marpコードをノートにコピペ

すぅさんプロンプト（Obsidian用に少しだけ改造）をノートにコピペ。すぅさんコードや別のmarpコードをそのまま貼り付けても大丈夫です。

```swift
---
marp: true
theme: custom
class: lead
paginate: true
header: " "
footer: " "
style: |-
  @import url('https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;700&display=swap');

  /* --- 色やフォントの基本設定 --- */
  :root {
    --color-background: #f8f8f4;
    --color-foreground: #3a3b5a;
    --color-heading: #4f86c6;
    --color-hr: #000000;
    --font-default: 'Noto Sans JP', 'Hiragino Kaku Gothic ProN', 'Meiryo', sans-serif;
  }

  /* --- スライド全体のスタイル --- */
  section {
    background-color: var(--color-background);
    color: var(--color-foreground);
    font-family: var(--font-default);
    font-weight: 400;
    box-sizing: border-box;
    border-bottom: 8px solid var(--color-hr);
    position: relative;
    line-height: 1.7;
    font-size: 22px;
    padding: 56px;
  }

  section:last-of-type {
    border-bottom: none;
  }

  /* --- 見出しのスタイル --- */
  h1, h2, h3, h4, h5, h6 {
    font-weight: 700;
    color: var(--color-heading);
    margin: 0;
    padding: 0;
  }

  /* タイトルページ(h1)のスタイル */
  h1 {
    font-size: 56px;
    line-height: 1.4;
    text-align: left;
  }

  /* 通常スライドのタイトル(##) */
  h2 {
    position: absolute;
    top: 40px;
    left: 56px;
    right: 56px;
    font-size: 40px;
    padding-top: 0;
    padding-bottom: 16px;
  }

  /* h2の疑似要素(::after)を使って、短い線を実装 */
  h2::after {
    content: '';
    position: absolute;
    left: 0;
    bottom: 8px;
    width: 60px;
    height: 2px;
    background-color: var(--color-hr);
  }

  /* h2と後続コンテンツの間のスペースを確保 */
  h2 + * {
    margin-top: 112px;
  }

  /* サブ見出し (例: 目的, 目標) */
  h3 {
    color: var(--color-foreground);
    font-size: 28px;
    margin-top: 32px;
    margin-bottom: 12px;
  }

  /* --- リストのスタイル --- */
  ol, ul {
    padding-left: 32px;
  }

  li {
    margin-bottom: 10px;
  }

  /* フッターとして機能する、太い青いラインを実装 */
  footer {
    font-size: 0;
    color: transparent;
    position: absolute;
    left: 56px;
    right: 56px;
    bottom: 40px;
    height: 8px;
    background-color: var(--color-heading);
  }

  /* ロゴの配置 */
  header {
    font-size: 0;
    color: transparent;
    background-image: url('ロゴ.png');
    background-repeat: no-repeat;
    background-size: contain;
    background-position: top right;
    position: absolute;
    top: 40px;
    left: calc(100% - 180px - 56px);
    width: 180px;
    height: 50px;
  }

  /* --- 特別なクラス --- */
  section.lead {
    border-bottom: 8px solid var(--color-hr);
  }

  /* タイトルページではフッターラインとロゴ(header)を非表示にする */
  section.lead footer,
  section.lead header {
    display: none;
  }

  section.lead h1 {
    margin-bottom: 24px;
  }

  section.lead p {
    font-size: 24px;
    color: var(--color-foreground);
  }
---
```

![画像](https://assets.st-note.com/img/1761389625-sPmXL420nTaiFrS6Y7fgB5EK.png?width=1200)

marpコードを貼り付けた状態

class: leadとpaginate: trueは必要なければ削除できますが、それ以外はスライドの表示に影響するため **削除しないことをお勧めします** 。

> **class: lead**: 最初のスライドにleadクラスを適用します。これは、タイトルページでフッターラインとロゴ（section.lead footer, section.lead header { display: none; }）を非表示にするためのカスタムスタイルと連携しています。削除すると、タイトルページにフッターとロゴが表示されるようになります。  
> **paginate: true**: スライドのページ番号を表示します。削除すると、ページ番号が表示されなくなります。

### STEP3：スライド化したいページを開いたままコマンド実行

スライド化したいページ（Markdown表記）を開いたままコマンドパレットを開き、「Slide Preview」を実行

![画像](https://assets.st-note.com/img/1761391136-1phw4CG8zQV3x72j9NAdYcOu.png?width=1200)

cmd+P（winはctrl+P）でコマンドパレットを起動できます
