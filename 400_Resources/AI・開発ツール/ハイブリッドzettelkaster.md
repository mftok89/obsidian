# ObsidianでのZettelkasten利用ガイド

## 1. Zettelkastenとは何か？

- **Zettelkastenの基本原則**: ノートの原子性、連結性、永続性。
- **メリット**: アイデアの発見、知識の体系化、創造性の向上。

## 2. ObsidianとZettelkastenの親和性

- **リンク機能**: `[[内部リンク]]`によるノート間の強力な連結。
- **グラフビュー**: 知識ネットワークの可視化。
- **タグと検索**: ノートの整理と迅速なアクセス。

## 3. ObsidianでのZettelkastenワークフロー

### 3.1. ノートの種類

- **リファレンスノート (文献ノート)**: 読んだ本や記事からの引用、要約。
- **フラットノート (アイデアノート)**: 独自のアイデア、考察、発見。これらがZettelkastenの核となる。
- **ハブノート (目次ノート)**: 特定のトピックに関するフラットノートをまとめる。

### 3.2. ノートの作成と命名規則

- **原子性**: 一つのノートには一つのアイデアのみ。
- **ユニークID**: Zettelkastenの伝統的なID（例: YYYYMMDDHHMMSS）や、Obsidianのファイル名（例: `具体的なアイデア名`）を利用。
- **明確なタイトル**: ノートの内容を簡潔に表すタイトル。

### 3.3. ノートの連結

- **双方向リンク**: `[[関連ノートのタイトル]]` を使って積極的にリンクを作成。
- **未作成リンク**: `[[まだ存在しないノートのアイデア]]` で将来のノートのプレースホルダーを作成。
- **バックリンクの活用**: 他のノートからの参照を確認し、関連性を深める。

### 3.4. タグとメタデータの活用

- **タグ**: `#トピック` や `#ステータス` などでノートを分類。
- **プロパティ**: YAMLフロントマターでソース、作成日、ステータスなどの構造化されたメタデータを追加。

## 4. Obsidian Zettelkastenをさらに活用するヒント

- **テンプレート**: ノート作成の効率化。
- **デイリーノート**: 新しいアイデアやタスクを日々記録し、後でフラットノートに展開。
- **プラグイン**:
    - **Dataview**: ノートをクエリして情報を整理。
    - **Longform**: より長い文章をZettelkasten方式で執筆。
    - **QuickAdd**: カスタムコマンドでワークフローを自動化。
