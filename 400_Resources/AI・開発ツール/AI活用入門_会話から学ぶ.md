会話に出てきたノウハウだけだと説明が飛んでしまう部分があるので、**初心者がつまずきやすいところには、基礎知識や周辺の話を講師が補足**しています。補足した部分には「📘 はじめての人へ」という囲みをつけました。会話そのものの話は「🗣 会話より」と書いてあります。

気になったところ、わかりにくかったところから読んでください。**全部を一度で理解しようとしなくて大丈夫**です。

## 1. そもそもAIって何？　最初の不安をなくそう

> 📘 **はじめての人へ**
>
> この章は会話には出てきませんが、ここを飛ばすと後がわかりにくいので、最初に少しだけ。

ここで言う「AI」とは、ChatGPTやClaude（クロード）などに代表される、**文章で会話できるAI**のことです。むずかしい正式名称は **「大規模言語モデル（LLM）」** と言いますが、名前は覚えなくて大丈夫。

イメージとしては、

> **「ものすごくたくさんの文章を読んで勉強した、とても物知りな相談相手」**

だと思ってください。あなたが文章（これを **プロンプト** ＝指示文 と呼びます）で話しかけると、AIが文章で返してくれます。

### 大事な心がまえ：AIは「ときどき間違える」

AIは物知りですが、**自信たっぷりに間違ったことを言う**ことがあります。これを専門用語で **「ハルシネーション（幻覚）」** と呼びます。

- 例：実在しない本のタイトルを「ありますよ」と答えてしまう
- 例：日付や数字をそれっぽく作ってしまう

だから、**大事なこと（仕事の数字、法律、健康など）は必ず自分でも確かめる**。これがAIと付き合う上での一番大切なルールです。「便利だけど、うのみにはしない」——この感覚さえあれば大丈夫です。

> 🗣 **会話より：** 熊澤さんも「作っている本人は間違いを見落とす」と話していました。AI自身も完璧ではないので、後で出てくる「別のAIにチェックさせる」工夫が活きてきます。

## 2. この会話で起きていたこと（全体像）

🗣 会話の中で、熊澤さんはAIに**「文字起こしの精度を上げるプログラム」を、自分の代わりに作らせていました**。

すごいのは、ただ作らせるだけでなく、次のような「かしこい使い方」をしていた点です。

- AIが作ったものを、**別のAIにチェックさせる**
- AIの失敗や学びを、**ノートアプリに記録して、どんどん賢くしていく**
- 個人情報が外に出ないよう、**自分のパソコンの中だけ**で動かす

その間、本人はお酒を飲んでリフレッシュ。AIが働いてくれるので、人間は見守るだけ。これが今のAIの便利さです。

> 📘 **はじめての人へ：プログラミングができなくても大丈夫**
>
> 「プログラムを作らせる」と聞くと身構えるかもしれませんが、熊澤さんは**コードを自分で書いていません**。日本語で「こうしてほしい」と頼んだだけです。
>
> 今のAIは、**ふつうの言葉でお願いするだけで、裏側の専門的な作業をAIがやってくれます**。だから「自分には無理」と思わなくて大丈夫。あなたがやるのは**「何をしてほしいか伝える」「結果を確認する」**——この2つだけです。

## 3. まず覚えたい基本のことば

むずかしそうな言葉が出てきますが、中身はとてもシンプルです。**4つだけ**おさえましょう。

### ① プロンプト（指示文）

> 📘 はじめての人へ

**AIへのお願いの文章**のことです。「〇〇について教えて」「この文章を要約して」——これが全部プロンプトです。

初心者がいちばん大事にすべきなのは、**プロンプトのコツ**です。3つだけ覚えてください。

1. **具体的に頼む** … ✕「いい感じにして」→ ◯「小学生にもわかるように、200字でまとめて」
2. **役割を与える** … 「あなたはプロの編集者です。その視点で直して」と言うと質が上がる
3. **一度でうまくいかなくて当たり前** … 「もっと短く」「例を入れて」と**追加で言い直せばOK**

> 💡 AIは会話の続きを覚えています。やり直しではなく「さっきのを、もっと〇〇して」と重ねていくのがコツです。

### ② ローカル / ローカルLLM

> 🗣 会話より

**「自分のパソコンの中だけで動くAI」**のことです。

ふだん使うAIは、インターネットの先にある会社のコンピューターに文章を送って、答えをもらいます。便利ですが、**大事な情報が自分の手元から外に出る**ことになります。

熊澤さんは福祉の仕事をしていて、**利用者さん（障害のある方）の名前など、外に出せない情報**をあつかいます。そこで「ローカルLLM」＝パソコンの中だけで完結するAIを使えば、情報が外にもれません。

> 📘 **はじめての人へ：最初はローカルでなくてOK**
>
> ローカルLLMは少し上級者向けです。**まずはChatGPTやClaudeなど、ふつうのAIで十分**。
> ただし、その場合は **「個人情報や会社の秘密は入力しない」** ことだけ守ってください。名前は「Aさん」に置きかえる、などの工夫で安全に使えます。「絶対に外に出せない情報を大量に扱う」段階になって初めて、ローカルを検討すればよいのです。

### ③ コンテキスト（AIの記憶力・集中力）

> 🗣 会話より

**「AIが今までの会話で、どれだけ頭を使ったか（覚えている量）」**のことです。

会話が長くなるほど、AIの「頭の中」はいっぱいになっていきます。会話の量が**ある程度を超えると、判断力が落ちてくる**と言われています。人間も、長時間しゃべり続けると集中力が切れますよね。それと同じです。

そんなときの対処は、

1. 「ここまでの作業内容の大事な部分だけ記録して」とAIに頼む
2. 会話を新しく始める（リセット）
3. 「さっきの記録の続きをお願い」と再開する

こうすると、AIは**最初からやり直さずに、しかも頭がスッキリした状態**で続けられます。

> 📘 **はじめての人へ：もっと手軽な合図**
>
> むずかしく考えず、**「話がかみ合わなくなってきたな」「さっき言ったことを忘れてるな」と感じたら、新しい会話を始める**。これだけでも十分です。話題がガラッと変わるとき（仕事の話 → 料理の話）も、新しい会話に分けると、AIが混乱しません。

### ④ プラン（いきなり実行させないモード）

> 🗣 会話より

**「いきなり実行しないで、まず計画だけ立ててもらうこと」**です。

AIは強力なので、「やって」と言うとどんどん作業を進めてしまいます。あとから不具合が出ると、元に戻すのが大変です。

そこで「**まず計画を立てて。実行はまだしないで**」と伝えると、AIは「こういう問題があります」「こう直す案があります」と**提案だけ**してくれます。それを見て納得してから「ではお願い」と言えば、失敗がぐっと減ります。

> 📘 **はじめての人へ：これは日常でも使える**
>
> 「プラン」は専門ツールだけの話ではありません。文章作成でも **「いきなり清書しないで、まず構成案を出して」** と頼むと、方向性を確かめてから進められて失敗しません。**「設計図を見てから建てる」** イメージです。

## 4. かしこい使い方 5つのコツ

ここからは、熊澤さんが実際にやっていた「上級のワザ」を、初心者向けにかみくだいて紹介します。**全部できなくてOK。「へえ、こんなことができるんだ」**と知るだけで十分です。

### コツ① AIにAIをチェックさせる（ダブルチェック）

🗣 これがこの会話の見どころです。

熊澤さんは、AIの中に**「ベテラン技術者」という別の人格のAI**をもう一人立ち上げ、最初のAIが作ったものを**チェックさせて**いました。この「別人格」を専門用語で **サブエージェント** と呼びます。

なぜわざわざ別人格に？　理由は人間と同じです。

> **作った本人は、自分のミスに気づきにくい。**
> でも、何も知らない別の人がパッと見ると「ここ、おかしくない？」と気づける。

> 📘 **はじめての人へ：道具がなくてもマネできる**
>
> サブエージェントは専門的な機能ですが、**考え方は今すぐマネできます。**
> 文章を書いてもらったら、**新しい会話を開いて**「この文章を、きびしい校正者の目でチェックして」と貼りつけるだけ。これだけで「作る人」と「直す人」を分けられ、ミスが減ります。**ダブルチェックをAIだけで回す**——これが本質です。

### コツ② 失敗を記録して、AIを育てる

🗣 熊澤さんは、AIが失敗（エラー）したとき、**その原因と対策をノートアプリに記録**していました。使ったアプリが **Obsidian（オブシディアン）** という、メモをためていくツールです。

記録をためると、

- AIが**同じ失敗をしにくくなる**
- 新しい失敗が起きても、**過去の記録を見て原因を早く見つけられる**

つまり、使えば使うほど**AIが経験から成長していく**のです。

> 📘 **はじめての人へ：あなたの「マイ・ルール帳」を作ろう**
>
> 同じことが、もっと簡単にできます。AIとのやりとりで「**このお願いの仕方だとうまくいった**」と思ったプロンプトを、メモ帳やスマホのメモに**コピーして保存**しておくのです。次から貼りつけて使えば、毎回ゼロから考えずにすみます。これがあなただけの「うまくいくお願い集」になります。

### コツ③ 「ひな形（テンプレート）」を用意しておく

🗣 毎回ゼロから説明するのは大変なので、熊澤さんは**自分の考え方や方針を書いた「ひな形」**を用意し、作業のたびにコピー＆ペーストしていました。

すると、こまかく説明しなくても、AIが**こちらの意図をくんで動いてくれます**。

おもしろいことに、方針を共有しておくと、AIのほうから
> 「あなたはシンプルなものを好みますが、今の指示だと複雑になりそうです。それでいいですか？」

と**逆に確認してくれる**こともあります。自分でも気づかなかった「考えのブレ」を指摘してもらえるわけです。

> 📘 **はじめての人へ：自己紹介メモを1つ作っておく**
>
> たとえば仕事で使うなら、こんな短いメモを用意して、会話の最初に貼るだけで効果があります。
>
> ```
> 【私について】
> ・福祉施設で働いています
> ・読み手は専門知識のない職員です
> ・むずかしい言葉は避け、やさしく短く書いてほしい
> ```
>
> たったこれだけで、AIの答えが毎回ぐっと自分好みになります。

### コツ④ 分からないことは「初心者向けに書き直して」と頼む

🗣 AIが作ったマニュアルに専門用語が多くて分からなかったとき、熊澤さんはこう頼んでいました。

> 「技術の初心者でもわかるように、細かい説明を入れて書き直して」
> 「私の環境（自分のアカウント名やフォルダ名）に合わせて、具体例を書きかえて」

すると、**自分の状況にぴったり合った、分かる説明**にしてくれます。

> 📘 **はじめての人へ：これは最強の魔法の言葉**
>
> 初心者がいちばん使うべきお願いがこれです。AIの説明が分からなくても、落ちこむ必要はゼロ。
> - 「**中学生でもわかるように説明して**」
> - 「**たとえ話を使って説明して**」
> - 「**専門用語を使わずに言いかえて**」
>
> 何度でも頼んでいいし、AIは嫌がりません。**分からないのはあなたのせいではなく、説明がむずかしすぎただけ**です。

### コツ⑤ AIを学ぶための学び方（インプットの工夫）

🗣 熊澤さんは、AIの使い方をこうやって独学していました。

1. 何かをしながら、AI関連の動画を**ながら見**で流しておく
2. 「使えそう」と思ったら、動画の**文字起こし機能**で文章にする
3. その文章を**AIに渡して「要約して」**と頼む
4. 要約から**大事なところだけ**を自分のメモにためる

一度で全部を理解しようとせず、**「流れ（イメージ）をつかむ」ことを優先**したのがコツだそうです。

> やり方の「流れ」がイメージできると、
> 「ここを乗りこえれば、こうできるはず」と思えて、最後までがんばれる。

> 📘 **はじめての人へ：完璧主義をすてよう**
>
> 初心者が挫折する一番の原因は **「全部わかってから使おう」** とすることです。でも実際は逆。**まず使ってみて、わからない言葉が出たらAIに聞く**——この順番のほうが何倍も速く身につきます。AIは、**AIの使い方を教えてくれる先生**でもあるのです。

## 5. 初心者がつまずきやすいポイントと対処法

> 📘 この章はまるごと講師からの補足です。多くの初心者が同じ場所でつまずきます。先に知っておくと安心です。

### つまずき①「最初の指示が短すぎて、思った答えと違う」

AIはエスパーではないので、**情報が少ないと当てずっぽうで答えます**。
→ **対処：** 「誰に向けて」「どんな形式で」「どれくらいの長さで」を一言そえる。
例：「**新人向けに・箇条書きで・5項目で**まとめて」

### つまずき②「一回で完璧を求めてしまう」

最初の答えが微妙でも失敗ではありません。**会話で育てるのが普通**です。
→ **対処：** 「もっと短く」「例を足して」「やわらかい口調で」と**言い直す**。

### つまずき③「AIの答えをそのまま信じてしまう」

前に書いたとおり、AIは自信たっぷりに間違えます（ハルシネーション）。
→ **対処：** 数字・日付・固有名詞・法律・健康の話は**自分でも確認**。心配なときは「**この情報は確かですか？　不確かな部分はどこですか？**」と聞き返すのも有効。

### つまずき④「個人情報・秘密を入力してしまう」

ふつうのAIに入れた情報は、自分の手元から外に出ます。
→ **対処：** 名前は「Aさん」に置きかえる。会社の秘密は入れない。**「外に出て困るものは入れない」**を徹底。

### つまずき⑤「会話が長くなって、AIが混乱する」

→ **対処：** 話題が変わったら**新しい会話を始める**。「コンテキスト」の章を参照。

### つまずき⑥「専門用語につまずいて、やる気をなくす」

→ **対処：** わからない言葉が出たら、**その場でAIに「〇〇ってどういう意味？」と聞く**。立ち止まらない。

## 6. 今日からできる最初の一歩

> 📘 はじめての人へ：知識を入れたら、あとは試すだけです。

いきなり熊澤さんのような上級ワザは不要です。**次の小さな練習から始めてみてください。**

1. **要約させる** … 長いメールや記事を貼って「3行でまとめて」
2. **言いかえさせる** … 自分の文章を貼って「もっとていねいな言い方にして」
3. **相談する** … 「〇〇について、初心者がまず知るべきことを5つ教えて」
4. **役割を与える** … 「あなたはベテランの〇〇です。その視点で助言して」
5. **チェックさせる** … 書いた文章を別の会話に貼って「おかしいところを指摘して」

この5つができれば、もう立派なAIユーザーです。あとは少しずつ「ローカル」「サブエージェント」などへ広げていけば大丈夫。

> **合言葉：「まず使う。わからなければ聞く。完璧を求めない。」**

## 7. 用語ミニ辞典

| ことば | やさしい意味 |
|---|---|
| **AI / LLM（大規模言語モデル）** | 文章で会話できる、とても物知りな相談相手 |
| **プロンプト** | AIへのお願いの文章（指示文） |
| **ハルシネーション** | AIが自信たっぷりに間違える現象。だから確認が必要 |
| **ローカル / ローカルLLM** | 自分のパソコンの中だけで動くAI。情報が外に出ない |
| **コンテキスト** | AIが今どれだけ覚えているか（記憶力・集中力の残り） |
| **プラン** | いきなり実行せず、計画・提案だけしてもらうこと |
| **サブエージェント** | AIの中に立ち上げる「別人格」。チェック役などに使う |
| **Obsidian（オブシディアン）** | メモ・知識をためていくノートアプリ |
| **Cursor（カーソル）** | プログラミングやサイト作成ができるAIツール |
| **GitHub CLI** | ファイルの保存・管理に使う、文字で操作する道具 |
