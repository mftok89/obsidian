## 🔒 Claude Code セキュリティ対策 6つのポイント

### 1. 機密ファイル（.env）を隠す

`.env`ファイルにはAPIキーやパスワードなど、絶対に流出してはいけない情報が含まれています。

- **対策:** Claude Codeに直接**「.envファイルを読み込まないように設定して」**と指示してください。AIが自動で禁止ルール（Deny rule）を作成してくれます。
    
- **理由:** プロンプトインジェクション（外部サイトを読み込んだ際に悪意のある指示を実行させられること）による情報漏洩を防ぐためです。
    

### 2. ファイル履歴（file-history）を削除する

Claude Codeは編集ミスに備え、自動でバックアップを保存しています。

- **場所:** `~/.claude/file-history`（Mac等の場合）
    
- **対策:** 定期的に**「ファイル履歴を削除して」**と依頼しましょう。
    
- **理由:** 過去にうっかり読み込ませてしまった機密情報が、バックアップとしてプレーンテキストで残ってしまうリスクがあるからです。
    

### 3. サンドボックス（Sandbox）を有効化する

AIの活動範囲を特定の「砂場（制限された環境）」の中に閉じ込めます。

- **対策:** ターミナルで **`/settings`** コマンドを打ち、設定画面からサンドボックスを**ON**にします。
    
- **理由:** 万が一AIが誤った操作をしても、PC全体への被害（全ファイル削除など）を防ぐ防波堤になります。
    

### 4. `CLAUDE.md` にルールを明文化する

プロジェクトごとに「守るべきルール」を覚えさせます。

- **対策:** **「セキュリティルールをCLAUDE.mdに追加して」**と依頼します。
    
- **内容:** 「.envは絶対に見ない」「外部へのデータ送信は必ず確認を求める」といった指示を書き込ませます。
    
- **理由:** セッション（会話）の開始時に必ず読み込まれるため、一貫したガードレールになります。
    

### 5. テスト駆動開発（TDD）を取り入れる

コードを書く前に「正解の基準」となるテストコードを先に作ります。

- **手順:**
    
    1. 「まず、この機能の**テストコード**を書いて」
        
    2. 「そのテストをパスするコードを実装して」
        
- **理由:** 意図しないバグ（セキュリティホール）の混入を防ぎ、AIが仕様通りに動いているかを客観的に確認できます。
    

### 6. コンテキスト（会話履歴）を定期的にクリアする

会話が長くなると、AIの「集中力（トークン）」が切れ、判断ミスをしやすくなります。

- **対策:** **`/clear`** コマンドで会話をリセットします。
    
- **コツ:** リセット前に**「今の進捗をドキュメントにまとめて」**と指示し、出力されたファイルを読み直させることで、高い精度（体力満タンの状態）で作業を再開できます。

## 💡 まとめ：初心者がまずやるべき一言

まずはClaude Codeを立ち上げたら、こう話しかけてみてください。

> 「**セキュリティ対策をしたいので、.envファイルを読み込まない設定と、CLAUDE.mdへのセキュリティルールの追加をお願いします。**」

これだけでも、何も対策をしない場合に比べて格段に安全性が高まります。

動画内でも触れられていましたが、AIに「ES（Yes）」を連打せず、**「今AIが何をしようとしているのか」を画面の確認メッセージでしっかりチェックする癖**をつけるのが最も大切な対策です。
