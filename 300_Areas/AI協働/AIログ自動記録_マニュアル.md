# AIログ自動記録 マニュアル

## 概要

Claude Code がすべてのプロジェクトで、以下の2種類のログを Obsidian に自動記録する仕組み。
設定日：2026-06-10

---

## 何が記録されるか

### 1. AIミス・教訓ログ

**記録先：** `300_Areas/AI協働/AIミス・教訓ログ.md`

**記録のタイミング：** Claude が以下のいずれかをしたとき（発生した時点で即時記録）

- 誤った実装・誤った情報を提示し、理さんに指摘された
- 実装したコードにバグがあり、修正が必要になった
- 思い込み・仮定の誤りで手戻りが発生した

**記録形式：**
```
## YYYY-MM-DD | プロジェクト名 | 一言タイトル
- **何が起きたか**:
- **原因の仮説**:
- **再発防止策**:
```

---

### 2. 決断ログ

**記録先：** `300_Areas/AI協働/決断ログ.md`

**記録のタイミング：** 理さんが設計・方針・技術選定などの意思決定をしたとき

**記録形式：**
```
## YYYY-MM-DD | プロジェクト名 | 決断タイトル
- **決断内容**:
- **検討した選択肢**:
- **決め手となった理由**:
```

---

## 仕組みの構成

### 設定ファイル

| ファイル | 役割 |
|---------|------|
| `~/.claude/CLAUDE.md` | 記録ルール本体。全プロジェクトで自動読み込み |
| `~/.claude/settings.json` | `NODE_EXTRA_CA_CERTS` で Obsidian の自己署名証明書を信頼設定済み |
| `~/.claude.json` | MCP サーバー `obsidian` をユーザースコープで登録済み |

### MCP 接続情報

- **エンドポイント：** `https://127.0.0.1:27124/mcp`
- **プロトコル：** HTTPS（自己署名証明書）
- **証明書保存先：** `~/.claude/certs/obsidian-local-rest-api.crt`
- **証明書有効期限：** 2027-06-10

---

## 「全プロジェクト共通」が効く理由

`~/.claude/CLAUDE.md` はユーザーレベルの設定ファイル。  
Claude Code は起動フォルダに関わらず、このファイルを必ず読み込む。  
そのため、`/home/osamu/projects/` 配下のどのフォルダで起動しても、また今後フォルダが増えても、追加設定なしで同じルールが適用される。

## トラブルシューティング

### 記録されなかった場合

Claude が「【完了】Obsidianに記録しました」と言わなかった場合は、「今のを記録して」と指示すれば即座に追記する。

### MCP 接続エラーが出た場合

1. Windows 側で Obsidian が起動しているか確認
2. Obsidian の「Local REST API」プラグインが有効になっているか確認
3. ターミナルで接続確認：
   ```bash
   curl -k https://127.0.0.1:27124/ -H "Authorization: Bearer <APIキー>"
   ```

### APIキーを再発行した場合（証明書も再生成される）

```bash
echo | openssl s_client -connect 127.0.0.1:27124 -servername 127.0.0.1 2>/dev/null \
  | openssl x509 -out ~/.claude/certs/obsidian-local-rest-api.crt
```

その後、`~/.claude.json` の Bearer トークンも新しい APIキーに更新する。
