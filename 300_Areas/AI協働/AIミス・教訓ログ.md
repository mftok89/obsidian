## 2026-06-11 | risk001 | tr:last-child が tbody 最終行にも適用されるCSSバグ
- **何が起きたか**: 月別×作業分類クロス集計で、合計行（tfoot）用の強調スタイルを `.work-month-table tr:last-child td` で指定していたため、tbody の最終行にも同じ色・太字が適用されていた。理さんに指摘されて発覚。
- **原因の仮説**: `:last-child` は「テーブル全体の最後の行」ではなく「親要素（tbody/tfoot それぞれ）の最後の tr」に一致するというCSS仕様の見落とし。
- **再発防止策**: 合計行など構造的に意味のある行は、位置セレクタではなく構造（tfoot）または専用クラスで指定する。

## 2026-06-11 | risk001 | 作業種類グラフだけ非文字列値で停止
- **何が起きたか**: 作業種類別件数グラフが「グラフ準備中」のまま表示されなかった。Excelの数式セルが数値(825)を返す行があり、`.trim()` で TypeError になっていた。
- **原因の仮説**: 他の関数には String() の型保証を入れていたのに、buildWorkTypeOption だけ対策が漏れていた（対策の適用が場当たり的だった）。
- **再発防止策**: 型の正規化は読み込み口（parseExcel）で一元的に行い、下流の各関数に個別対策を散らばらせない。今回その方針で修正済み。

## 2026-06-11 | risk001 | MCP設定調査時に認証トークンをチャットに表示
- **何が起きたか**: Obsidian MCP の接続調査で設定ファイル（.claude.json）の中身をそのまま出力し、Local REST API の Bearer トークンがチャットに表示された。
- **原因の仮説**: 「機密情報をチャットに表示しない」ルールを .env ファイル限定と狭く解釈し、設定ファイルにも認証情報が含まれる可能性を考慮せずダンプした。
- **再発防止策**: 設定ファイルを表示する際は headers / token / key / password 系のフィールドをマスクしてから出力する。ローカル専用キーでも扱いは同じにする。


## 2026-06-17 | 00format01code | ワークフロー手順を複数ファイルに重複させた
- **何が起きたか**: 各設定（CLAUDE.md/orchestrator.md/.cursorrules）にSDD+TDD手順の実体を書き込み、開発手順.mdと内容が4ファイルに重複。シニアエンジニアのサブエージェントレビューで「設定ドリフトを生む」「レビュー順序が3通りに食い違う」と指摘された。
- **原因の仮説**: 各エージェントが確実に手順を読むよう、各ファイルに手順を厚く書きすぎた。DRY（単一の正本）の観点が抜けていた。
- **再発防止策**: 手順・ルールは正本(SSoT)を1ファイルに定め、他は「参照＋役割固有の注意点」に限定する。プラン段階でレビューに出していれば事前に防げた（＝設計フェーズのレビューの価値）。
