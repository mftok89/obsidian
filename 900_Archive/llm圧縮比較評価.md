# LLM圧縮テスト 比較評価

> 作成日: 2026-07-01
> 対象: `検討項目とideaツール組み合わせ/` 配下の4ファイル（000・001・003・004）
> 目的: 同一プロンプトを4条件（日本語/英語 × 生コンテキスト/LLM圧縮コンテキスト）で実行した結果を比較し、コスト・速度・出力品質への影響を評価する。

---

## 0. 実験の正体

4ファイルは単なる4案ではなく、**同一プロンプトを4条件で実行した比較実験**の成果物。`LLM圧縮テスト.txt` のログと `test0xx_*` フォルダの内容から特定・裏付け済み（`diff` で完全一致を確認）。

| ファイル | 条件 | 由来フォルダ |
|---|---|---|
| 000 | 日本語・生コンテキスト | `test000_japanese` |
| 001 | 英語・生コンテキスト | `test001_english` |
| 003 | 日本語・LLM圧縮コンテキスト | `test003_japanese_llm` |
| 004 | 英語・LLM圧縮コンテキスト | `test004_english_llm` |

「LLM圧縮」＝ `CLAUDE.md` や `000_me` をカタカナ語・英語混じりの短縮表現に圧縮したもの（例: 「役割」→「Role」、業務用語を英語のまま残す等）。`idea/` フォルダは4条件とも同一（28K不変）で、差は `000_me` と `CLAUDE.md` の圧縮度合いのみ。

プロンプト自体（`LLM圧縮テスト.txt` 1〜5行目）:
> 000_meフォルダと、ideaフォルダを読み込んで、これから私が検討したほうがいい項目と、その際に利用するideaツールの組み合わせの候補を、10個上げてください。
> そして、そこにあなたが推奨するランキングと、その理由も添えてください。
> この内容を、MDファイルにまとめてください。

## 1. コスト・速度（定量）

| | 000 (JP生) | 001 (EN生) | 003 (JP圧縮) | 004 (EN圧縮) |
|---|---|---|---|---|
| コスト | $0.63 | $0.71 | $0.70 | $0.74 |
| 壁時計時間 | 11m7s | 6m26s | 6m23s | 4m54s |
| API時間 | 2m13s | 2m37s | 2m29s | 2m19s |
| cache read | 283.8k | 216.7k | 248.1k | 273.5k |
| cache write | 28.4k | 34.6k | 32.5k | 36.8k |
| 出力行数 | 73行 | 90行 | 52行 | 37行 |

**重要な逆説**: 圧縮するほどコストが下がるどころか**上昇**している（$0.63 → $0.74）。`000_me` + `CLAUDE.md` の圧縮で削れたのはせいぜい数KB〜十数KBで、cache read全体（200k〜280k超）に対して比率が小さすぎ、削減効果が誤差に埋もれている。壁時計時間の短縮（11分→5分）はAPI時間がほぼ同じ（2m13s〜2m37s）であることから、圧縮そのものというより実行間のばらつき（thinking/ツール呼び出しの揺らぎ）である可能性が高い。

### 最大の発見: 圧縮スタイルの「感染」

日本語/英語という条件軸ではなく、**「生」か「LLM圧縮」か**が出力の文体を決めている。000（JP生）と001（EN生）はどちらも自然な日本語散文だが、003と004（どちらも圧縮コンテキスト条件）はどちらも英語・カタカナ語が混入した読みにくい文体になっている。

つまり**圧縮スタイル（カタカナ短縮語）がモデル自身の出力文体にまで「感染」し、成果物の日本語品質を下げる副作用**が明確に確認できる。これはコスト削減効果がほぼゼロだったことと合わせて、この圧縮手法のデメリットの方が大きいことを示唆する結果。

## 3. 総合評価

- **1位: 001（英語・生）** — コンテンツの質だけで見て頭一つ抜けている。ランキング表が最初から順位順、施策の「狙い」明記、情報源の出典区別、次の一手の提示と、実務で使える完成度が高い。
- **2位: 000（日本語・生）** — 文体は綺麗だが、A-J記号とランキングの往復が必要な構成がやや不親切。
- **3位: 004（英語・圧縮）** — トークン消費最小・行数最少ながら、MVP分解やメタ原則の遵守など密度の高い洞察を含む点は評価できるが、文体の乱れが実用上マイナス。
- **4位: 003（日本語・圧縮）** — 004と近い傾向だが独自性がやや弱く、文体の乱れも同程度で相対的に見劣りする。

### 実験としての結論

この4条件比較では、LLM圧縮はコスト削減に寄与せず、むしろ出力の日本語品質を損なう副作用が観測された。「英語での指示（001）」が最もコストパフォーマンス良く高品質な結果を生んでおり、圧縮より「英語プロンプト化」の方が有望という示唆が得られる。
